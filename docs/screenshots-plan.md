# Screenshots Plan

## Recommended Screenshots for Portfolio

### 1. Dashboard Home — Full View
**What to capture**: Full dashboard with KPI cards, revenue chart, growth chart, activity feed, alerts, and quick actions.
**Why**: Shows the complete product at a glance. Demonstrates data visualization, layout design, and information hierarchy.

### 2. Dashboard Home — KPI Cards Close-up
**What to capture**: The 6 KPI metric cards with trend indicators.
**Why**: Demonstrates attention to detail — typography, color-coded trends, compact data display.

### 3. Accounts Table — Full View
**What to capture**: The accounts page with summary cards, search bar, filters, and the data table populated with accounts.
**Why**: Data tables are a core SaaS pattern. Shows filtering, badges, avatars, and professional table design.

### 4. Account Detail Drawer
**What to capture**: The slide-over drawer showing a single account's full details.
**Why**: Demonstrates interactive UI patterns — drawers, detail views, action buttons.

### 5. Workflows Page
**What to capture**: The task cards grid with health summary, priority badges, and status indicators.
**Why**: Shows card-based UI, workflow visualization, and a different layout pattern from the table views.

### 6. Billing Dashboard
**What to capture**: Revenue metrics, donut chart, top accounts bar chart, and invoices table.
**Why**: Financial dashboards are high-value. Shows chart variety and monetization-focused design.

### 7. Settings — Organization Tab
**What to capture**: The settings page with tab navigation, form fields, and billing plan card.
**Why**: Demonstrates form design patterns and admin UX.

### 8. Settings — Integrations Tab
**What to capture**: The integrations list with connected/disconnected states.
**Why**: Shows realistic SaaS integration patterns.

### 9. Mobile View — Dashboard
**What to capture**: Dashboard on mobile viewport with collapsed sidebar.
**Why**: Demonstrates responsive design capability.

### 10. Mobile View — Sidebar Open
**What to capture**: Mobile view with sidebar overlay open.
**Why**: Shows mobile navigation pattern with smooth overlay.

## Screenshot Tips

- Use a clean browser window (no bookmarks bar, extensions)
- Set browser to 1440x900 for desktop shots
- Set browser to 375x812 for mobile shots
- Use light mode (the default)
- Ensure all mock data is visible and realistic
